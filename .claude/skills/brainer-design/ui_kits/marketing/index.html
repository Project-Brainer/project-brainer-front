Primitives

Five node types. Infinite systems.

Everything in a product reduces to a few primitives. Give them shape on the canvas and let the arrows tell the story.

Screens

Each screen declares its route and UI elements. Connect them to describe navigation.

Data models

Entities with typed fields. String, number, date, file, boolean, enum. No ambiguity.

API endpoints

Method plus path. Wire endpoints to models for reads, writes, and updates.

Roles

Who sees what? Attach roles to screens and endpoints to describe access.

UI elements

Buttons, forms, lists — placed inside screens. Every element can trigger an action.

Connections

Five types: opens, calls API, reads data, writes data, updates. Labeled on every arrow.

Simulation

Click through your logic before you build it.

Flip to Simulation and the canvas becomes a clickable prototype. Mock data flows, API calls log in real time, navigation animates between screens.
